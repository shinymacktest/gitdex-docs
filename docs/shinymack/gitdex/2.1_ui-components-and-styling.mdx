# UI Components and Styling

This section documents the reusable UI components and the styling approach used within the client application of GitDex. GitDex aims to provide a consistent and visually appealing user experience, and this is achieved through a combination of Tailwind CSS, custom CSS variables, and reusable React components.

## Styling Approach

GitDex utilizes Tailwind CSS for utility-first styling, complemented by custom CSS variables defined in `client/src/app/globals.css`. This approach allows for a balance between rapid prototyping and maintainable, themeable styles.

## UI Components

GitDex leverages reusable React components to maintain a consistent look and feel throughout the application. Key components include buttons, cards, and the navigation bar.

### Button Component (`client/src/components/ui/button.tsx`)

The `Button` component is a fundamental UI element, providing various styles (variants) and sizes. It utilizes the `class-variance-authority` library (`cva`) to manage different visual states based on props.

```typescript
import * as React from "react"
import { Slot } from "@radix-ui/react-slot"
import { cva, type VariantProps } from "class-variance-authority"

import { cn } from "@/lib/utils"

const buttonVariants = cva(
  "inline-flex items-center justify-center whitespace-nowrap rounded-md text-sm font-medium ring-offset-background transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:pointer-events-none disabled:opacity-50",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        destructive:
          "bg-destructive text-destructive-foreground hover:bg-destructive/90",
        outline:
          "border border-input bg-background hover:bg-accent hover:text-accent-foreground",
        secondary:
          "bg-secondary text-secondary-foreground hover:bg-secondary/80",
        ghost: "hover:bg-accent hover:text-accent-foreground",
        link: "text-primary underline-offset-4 hover:underline",
      },
      size: {
        default: "h-10 px-4 py-2",
        sm: "h-9 rounded-md px-3",
        lg: "h-11 rounded-md px-8",
        icon: "h-10 w-10",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
)

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    const Comp = asChild ? Slot : "button"
    return (
      <Comp
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    )
  }
)
Button.displayName = "Button"

export { Button, buttonVariants }
```

[View on GitHub](https://github.com/shinymack/gitdex/blob/main/client/src/components/ui/button.tsx)

The `cva` function allows defining different styles based on the `variant` and `size` props.  The `cn` utility (likely from `classnames` or similar) is used to conditionally apply CSS classes.  The `asChild` prop uses `Radix UI`'s `Slot` component to allow the `Button` to render as a different HTML element.

### Card Component (`client/src/components/ui/card.tsx`)

The `Card` component provides a container for grouping related content. It includes sub-components for the header, title, description, content, and footer.

```typescript

import { cn } from "@/lib/utils"

const Card = React.forwardRef<
  HTMLDivElement,
  React.HTMLAttributes<HTMLDivElement>

  <div
    ref={ref}
    className={cn(
      "rounded-lg border bg-card text-card-foreground shadow-sm",
      className

  />
))
```

### Floating Navigation Bar (`client/src/components/navbar.tsx`)

The `FloatingNav` component provides the main navigation for the application. It includes links to key sections, a GitHub link, and a mobile menu.

```typescript
'use client';

import { useState, useEffect } from 'react';
import { Github, BookOpen, Menu, X } from 'lucide-react';
import Link from 'next/link';
import { Button } from '@/src/components/ui/button';

export function FloatingNav() {
  const [mounted, setMounted] = useState(false);
  const [mobileMenuOpen, setMobileMenuOpen] = useState(false);

  useEffect(() => {
    setMounted(true);
  }, []);

  if (!mounted) return null;

  return (
    <nav
      className={`
        fixed top-4 left-1/2 -translate-x-1/2 z-50
        w-[90%] md:w-[80%] lg:w-[70%]
        rounded-2xl border border-border/40
        bg-background/50 backdrop-blur-[5px]
        shadow-md
        transition-all duration-300
      `}
    >
      <div className="px-4 py-3 flex items-center justify-between">
        <Link href="/" className="flex items-center gap-2">
          <span className="text-xl">GitDex</span>
        </Link>

        <div className="flex items-center gap-4">
          {/* Desktop Navigation */}
          <div className="hidden md:flex items-center gap-4">
            <a
              href="https://github.com/shinymack/gitdex"
              target="_blank"
              rel="noopener noreferrer"
            >
              <Button variant="ghost" size="sm">
                <Github className="w-4 h-4" />
              </Button>
            </a>
          </div>

          {/* Mobile Menu Button */}
          <Button
            variant="ghost"
            size="sm"
            className="md:hidden"
            onClick={() => setMobileMenuOpen(!mobileMenuOpen)}
          >
            {mobileMenuOpen ? <X className="w-5 h-5" /> : <Menu className="w-5 h-5" />}
          </Button>
        </div>
      </div>

      {/* Mobile Menu */}
      {mobileMenuOpen && (
        <div className="md:hidden border-t border-border/40 bg-background/80 backdrop-blur-lg rounded-b-2xl">
          <div className="px-4 py-3 flex flex-col gap-3">
            <a
              href="https://github.com/shinymack/gitdex"
              target="_blank"
              rel="noopener noreferrer"
              onClick={() => setMobileMenuOpen(false)}
            >
              <Button variant="ghost" size="sm" className="w-full justify-start gap-2">
                <Github className="w-4 h-4" />
                <span>GitHub</span>
              </Button>
            </a>
          </div>
        </div>
      )}
    </nav>
  );
}
```

[View on GitHub](https://github.com/shinymack/gitdex/blob/main/client/src/components/navbar.tsx)

The `FloatingNav` component uses Tailwind CSS classes for positioning, sizing, and styling.  It also uses React's `useState` and `useEffect` hooks to manage the component's state and handle mounting. The mobile menu is conditionally rendered based on the `mobileMenuOpen` state.

## Key Integration Points

- **Consistent Styling:** The use of CSS variables in `globals.css` ensures that colors, fonts, and other design elements are consistent across the application.
- **Reusable Components:** The `Button` and `Card` components are designed to be reusable, reducing code duplication and improving maintainability.
- **Responsive Design:** The `FloatingNav` component includes a mobile menu, ensuring that the application is usable on a variety of screen sizes.
- **Theming**: Although theme switching functionality is not in the code, the dark and light theme definition allows the application to easily switch between themes.

## Best Practices

- **Follow the DRY principle:** Avoid repeating styles by creating reusable components and using CSS variables.
- **Use a consistent naming convention:** Follow a consistent naming convention for CSS classes and component names.
- **Write clear and concise code:** Write code that is easy to understand and maintain.
- **Test your components:** Test your components to ensure that they work as expected.

By following these best practices, you can create a UI that is both visually appealing and easy to maintain.
