## Access Control & Analytics

### Update File Settings
Update the access restrictions, expiration dates, and security settings for a specific file.

- **Endpoint:** `/api/analytics/set`
- **Method:** `POST`
- **Content-Type:** `application/json`

#### Request Body
| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `file_id` | `string` | Yes | The unique identifier of the file to update. |
| `maxViews` | `number` | No | Maximum allowed views before restriction. |
| `maxDownloads` | `number` | No | Maximum allowed downloads before restriction. |
| `expiresAt` | `string` | No | ISO 8601 timestamp for file expiration. |
| `password` | `string` | No | Password required to access the file. |
| `deleteOnExpire` | `boolean` | No | If `true`, file is marked for deletion upon expiration. |
| `deleteOnLimit` | `boolean` | No | If `true`, file is marked for deletion when limits are reached. |

#### Responses
- **200 OK**: `{"success": true, "message": "Access controls updated"}`
- **500 Internal Server Error**: `{"success": false, "error": "Error message"}`

## File Management

### Delete File Pipeline
Performs a coordinated deletion of a file from physical storage and updates the database state.

- **Endpoint:** `/api/deletepipeline`
- **Method:** `DELETE`
- **Content-Type:** `application/json`

#### Request Body
| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `file_id` | `string` | Yes | The unique identifier of the file to be deleted. |

#### Workflow
1. **Lookup**: Validates existence and retrieves `file_key` from Supabase.
2. **Storage Purge**: Deletes the object from the AWS S3 bucket.
3. **State Update**: Sets `is_active` to `false` and updates `expires_at` to the current timestamp in the database.

#### Responses
- **200 OK**: `{"success": true, "message": "File deleted from S3 and marked inactive in DB"}`
- **400 Bad Request**: `{"success": false, "error": "file_id is required"}`
- **404 Not Found**: `{"success": false, "error": "File not found"}`
- **500 Internal Server Error**: `{"success": false, "error": "Failed to delete file from S3", "details": "..."}`

## Utilities

### Test S3 Upload
A diagnostic endpoint used to verify AWS S3 connectivity and permissions.

- **Endpoint:** `/api/test`
- **Method:** `POST`
- **Content-Type:** `multipart/form-data`

#### Request Body
| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `file` | `File` | Yes | The binary file to upload for testing. |

#### Responses
- **200 OK**: `{"success": true, "url": "https://bucket.s3.region.amazonaws.com/test/timestamp-name"}`
- **400 Bad Request**: `{"error": "No file uploaded"}`
- **500 Internal Server Error**: `{"error": "S3 Upload Error message"}`
