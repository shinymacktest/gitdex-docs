## Introduction
This section delves into the primary pages of the Chatty frontend application, focusing on `HomePage.jsx` and `LoginPage.jsx`. These pages serve as the main entry points for authenticated and unauthenticated users, respectively, orchestrating various components and interacting with global state management.

## Home Page
The `HomePage.jsx` component is the central hub for authenticated users, displaying the main chat interface. It conditionally renders different components based on the application's state, primarily driven by the `useChatStore`.

### Structure and Logic
The `HomePage` dynamically decides whether to show a placeholder `NoChatSelected` component (when no conversation is active) or the `ChatContainer` (when a user is selected for chat). It also integrates `Sidebar` for user listing and `FriendsBox` (if implemented and `isFriendsBoxOpen` is true).

```jsx
// frontend/src/pages/HomePage.jsx
import ChatContainer from "../components/ChatContainer";
import FriendsBox from "../components/FriendsBox";
import NoChatSelected from "../components/NoChatSelected";
import Sidebar from "../components/Sidebar";
import { useChatStore } from "../store/useChatStore"

const HomePage = () => {
  const { selectedUser, isFriendsBoxOpen } = useChatStore();
  return (
      <div className="h-screen bg-base-200">
      <div className="flex items-center justify-center pt-20 px-4 w-full">
        <div className="bg-base-100 rounded-lg shadow-xl w-full max-w-6xl h-[calc(100vh-8rem)]">
          <div className="flex h-full rounded-lg overflow-hidden w-full">
            <Sidebar />
            {!selectedUser ? <NoChatSelected /> : <ChatContainer />}
            {isFriendsBoxOpen && <FriendsBox />}
          </div>
        </div>
      </div>
    </div>
  )
}

export default HomePage
```

## Login Page
The `LoginPage.jsx` handles user authentication, providing options for email/password login and Google OAuth. It manages form state, displays loading indicators, and integrates with the `useAuthStore` for authentication logic.

### Authentication Flow
Users can sign in using their email and password. The page includes input fields with validation hints, a password visibility toggle, and a submit button that disables during the login process. Additionally, there's a "Sign in with Google" option, linking to the backend's Google authentication route.
