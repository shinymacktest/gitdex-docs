# Settings and Theme Management

<TOC />

The user interface of the application provides a dedicated settings page (`SettingsPage.jsx`) where users can customize their chat experience, primarily by selecting different themes. This page integrates with a global theme state management system to apply changes across the application.

## 2.2.1 Settings Page (`SettingsPage.jsx`)

The `SettingsPage` component is a React page that allows users to change the application's visual theme. It fetches available themes from a constant and displays them as interactive buttons.

### Theme Selection Interface
The page renders a grid of theme buttons. Each button displays a small preview of the theme's primary, secondary, accent, and neutral colors. When a user clicks a button, the `setTheme` function from `useThemeStore` is invoked to update the active theme. The currently selected theme is highlighted.

```jsx
// From frontend/src/pages/SettingsPage.jsx
const { theme, setTheme } = useThemeStore();

return (
  <div className="h-screen container mx-auto px-4 pt-20 max-w-5xl">
    {/* ... other UI ... */}
    <div className="grid grid-cols-4 sm:grid-cols-6 md:grid-cols-8 gap-2">
      {THEMES.map((t) => (
        <button
          key={t}
          className={`
            group flex flex-col items-center gap-1.5 p-2 rounded-lg transition-colors
            ${theme === t ? "bg-base-200" : "hover:bg-base-200/50"}
          `}
          onClick={() => setTheme(t)} // Updates the theme state
        >
          <div
            className="relative h-8 w-full rounded-md overflow-hidden"
            data-theme={t} // Applies the theme for preview
          >
            {/* Theme color previews */}
            <div className="absolute inset-0 grid grid-cols-4 gap-px p-1">
              <div className="rounded bg-primary"></div>
              <div className="rounded bg-secondary"></div>
              <div className="rounded bg-accent"></div>
              <div className="rounded bg-neutral"></div>
            </div>
          </div>
          <span className="text-[11px] font-medium truncate w-full text-center">
            {t.charAt(0).toUpperCase() + t.slice(1)}
          </span>
        </button>
      ))}
    </div>
    {/* ... chat preview section ... */}
  </div>
);
```
[Source: `frontend/src/pages/SettingsPage.jsx`](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/src/pages/SettingsPage.jsx)

### Live Chat Preview
The `SettingsPage` includes a static chat UI preview that dynamically updates its appearance based on the selected theme. This allows users to visualize how each theme will look in a typical chat context before applying it permanently. The preview uses mock messages and a read-only input field.

The preview uses `data-theme={t}` attribute on the theme buttons and applies `data-theme` to elements within the preview section, allowing the `daisyUI` themes to instantly change the appearance.

```jsx
// From frontend/src/pages/SettingsPage.jsx
const PREVIEW_MESSAGES = [
  { id: 1, content: "Hey! How's it going?", isSent: false },
  {
    id: 2,
    content: "I'm doing great! Just working on some new features.",
    isSent: true,
  },
];
// ...
<div className="rounded-xl border border-base-300 overflow-hidden bg-base-100 shadow-lg">
  <div className="p-4 bg-base-200">
    <div className="max-w-lg mx-auto">
      {/* Mock Chat UI with theme-dependent classes */}
      <div className="bg-base-100 rounded-xl shadow-sm overflow-hidden">
        {/* ... Chat Header, Messages, Input with dynamic classes based on theme ... */}
      </div>
    </div>
  </div>
</div>
```

## 2.2.2 Theme Selection Workflow

The process of changing a theme is straightforward: the `SettingsPage` component subscribes to the `useThemeStore`, and user interactions directly trigger state updates in the store, which in turn re-renders the UI with the new theme.
