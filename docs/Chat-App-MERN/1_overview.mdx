## Authentication Flow

The authentication process involves two main pages: `SignUpPage.jsx` and (presumably) a `LoginPage.jsx` (not provided in this code chunk).  The `SignUpPage.jsx` handles user registration, while the `LoginPage.jsx` (inferred) would handle user login.  Both interact with the backend via the `useAuthStore.js` which uses axios for API calls.

The `useAuthStore.js` manages the authentication state using Zustand, a lightweight state management solution.  Key functions include `signup`, `login`, `logout`, and `checkAuth`.

```javascript
// Snippet from useAuthStore.js
signup: async (data) => {
    set({ isSigningUp: true });
    try {
        const res = await axiosInstance.post("/auth/signup", data);
        set({ authUser: res.data });
        toast.success("Account created successfully");
        get().connectSocket();
    } catch (error) {
        toast.error(error.response.data.message);
    } finally {
        set({ isSigningUp: false });
    }
},
```
