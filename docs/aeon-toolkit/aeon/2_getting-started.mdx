# Getting Started

Welcome to `aeon`, a toolkit for time series analysis in Python. This guide will walk you through the installation process and help you set up `aeon` for your initial projects.

## Installation Options

`aeon` supports Python versions 3.9, 3.10, 3.11, and 3.12. Ensure you have a compatible version of Python installed before proceeding. `aeon` is tested on macOS, Ubuntu, and Windows.

There are three primary ways to install `aeon`:

*   Install the latest release from PyPi (Recommended).
*   Install the latest release from conda-forge.
*   Install the latest development version from GitHub via pip.

### Optional Dependencies

`aeon` has core dependencies and optional dependencies. Installing all dependencies allows you to use all `aeon` functionality without interruption but can increase installation time and introduce versioning constraints.

## Installation via PyPi (Recommended)

We recommend using a virtual environment to avoid conflicts with other Python packages.

### Using a pip venv

Create a virtual environment:

```bash
python -m venv aeon-venv #Windows and macOS
python3 -m venv aeon-venv #Linux
```

Activate the environment:

```bash
aeon-venv\Scripts\activate #Windows
source aeon-venv/bin/activate #macOS and Linux
```

Install `aeon` with core dependencies:

```bash
pip install -U aeon
```

Install `aeon` with all stable dependencies:

```bash
pip install -U aeon[all_extras]
```

[View on GitHub](https://github.com/aeon-toolkit/aeon/blob/main/pyproject.toml)

**Note:** Some shells require quotes around `[all_extras]`:

```bash
pip install -U aeon"[all_extras]"
```

**Warning:** Some dependencies in `all_extras` may cause installation issues on macOS with ARM processors. See the troubleshooting section below.

Verify the installation:

```bash
pip show aeon  # See information about the installation i.e. version and file location
pip freeze  # See all installed packages for the current environment
```

## Installation via conda-forge

Create a new environment and install `aeon`:

```bash
conda create -n aeon-env -c conda-forge aeon
conda activate aeon-env
```

Verify the installation:

```bash
conda list aeon  # See information about the installation i.e. version and file location
conda list  # See all installed packages for the current environment
```

Optional dependencies must be installed separately with `conda`.

## Install the latest development version using pip

Uninstall any existing `aeon` installations:

```bash
pip uninstall aeon
```

Install directly from the `aeon` GitHub `main` branch:

```bash
pip install -U git+https://github.com/aeon-toolkit/aeon.git@main
```

Install from GitHub `main` branch with all stable dependencies:

```bash
pip install -U "aeon[all_extras] @ git+https://github.com/aeon-toolkit/aeon.git@main"
```

The `.readthedocs.yml` file configures the documentation build process:

```yaml

  install:

  os: ubuntu-24.04
  tools:
    python: "3.11"

sphinx:
  configuration: docs/conf.py
```

The above configuration uses Ubuntu 24.04, Python 3.11, and installs the documentation dependencies from the `docs` extra requirements. This ensures the documentation builds correctly on Read the Docs.

## Troubleshooting

### `ModuleNotFoundError`

This error typically occurs when using an estimator that requires a package not installed. Install the missing package, or install `aeon` with `all_extras`.

### `ImportError`

This error often indicates an improperly linked virtual environment. Ensure your environment is activated and linked to your IDE.

### Installing `all_extras` on Mac with an ARM processor

Some libraries in `all_extras` may not be compatible with ARM-based processors. Try installing soft dependencies separately.

### `no matches found` when installing `all_extras`

Some shells treat square brackets as special characters. Surround the dependency portion with quotes:

```bash
pip install -U aeon"[all_extras]"
```

## Key Integration Points

When integrating `aeon` into your projects, consider the following:

*   **Virtual Environments**: Always use virtual environments to manage dependencies.
*   **Dependency Management**: Choose between core and optional dependencies based on your project's needs.
*   **macOS ARM Compatibility**: Be aware of potential issues with `all_extras` on macOS ARM processors.

By following these guidelines, you can ensure a smooth and efficient setup of `aeon` for your time series analysis tasks.
