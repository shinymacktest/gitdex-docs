# Introduction

GitDex is a sophisticated tool designed to transform any GitHub repository into beautiful, AI-powered interactive documentation within seconds [README.md:3-5](). By automating the bridge between raw source code and human-readable documentation, GitDex provides a search-ready web reader integrated with an interactive AI chat assistant for deep repository analysis [README.md:7-8]().

## Primary Value Proposition

The core value of GitDex lies in its ability to eliminate the manual overhead of maintaining technical documentation. Instead of manually writing guides, GitDex employs an AI-driven pipeline to analyze codebase structures, plan an organized Table of Contents (TOC), and generate comprehensive Markdown documentation using Large Language Models (LLMs) [README.md:7-8]().

### Key Capabilities

*   **Automated Indexing**: A high-performance, multi-step pipeline that scans the repository, plans the documentation structure, and writes the content [README.md:12-13]().
*   **AI-Driven Interaction**: An AI code assistant that utilizes manual ReAct loops to provide accurate answers to complex questions about the repository [README.md:14-15]().
*   **Architectural Visualization**: The system automatically generates Mermaid diagrams to help users visualize the codebase architecture [README.md:16-17]().
*   **Serverless Scalability**: To overcome serverless execution timeout limits, GitDex uses a custom-built queue system powered by Upstash Redis and QStash to decouple the indexing workflow [README.md:18-19]().

## High-Level System Overview

GitDex is architected as a split-package system consisting of a dedicated backend orchestrator and a frontend presentation layer [README.md:27-31]().

### GitDex Server
The server acts as the background document indexing pipeline and API server [server/README.md:3-4](). Its primary responsibilities include:
*   Scanning GitHub repositories using Octokit [server/README.md:8-9]().
*   Orchestrating the TOC planning and step-by-step document generation via Google Gemini [server/README.md:3-8]().
*   Managing the QStash and Upstash Redis queue to ensure reliable execution in serverless environments [server/README.md:8-9]().
*   Committing the final generated documentation directly back to the user's GitHub docs repository [server/README.md:3-8]().

### GitDex Client
The client is an interactive documentation viewer and chat assistant [client/README.md:3-4](). Its primary responsibilities include:
*   Rendering codebase documentation as searchable pages using Fumadocs [client/README.md:3-7]().
*   Providing the user interface for the AI chatbot, allowing users to converse directly with the repository content [client/README.md:3-7]().
