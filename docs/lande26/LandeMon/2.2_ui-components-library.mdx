# UI Components Library

This section provides an overview of the custom UI components available in LandeMon, along with guidance on their usage. These components are designed for consistency and ease of integration within your applications.

## Button

The `Button` component is a versatile element for user interactions. It supports various visual styles and sizes, controlled through props.

### Variants

- `default`: The standard primary button style.
- `destructive`: For actions that have significant consequences.
- `outline`: A bordered button, often used for secondary actions.
- `secondary`: A less prominent style for secondary actions.
- `ghost`: A minimal button that appears on hover.
- `link`: A text-based link that resembles a button.

### Sizes

- `default`: Standard button size.
- `sm`: Small button size.
- `lg`: Large button size.
- `icon`: For icon-only buttons.

### Usage Example

```tsx
import { Button } from "@/components/ui/button";

function ExampleComponent() {
  return (
    <div className="flex gap-2">
      <Button>Default Button</Button>
      <Button variant="destructive" size="lg">
        Destructive Large
      </Button>
      <Button variant="outline" size="sm">
        Outline Small
      </Button>
    </div>
  );
}
```

## Dialog

The `Dialog` component provides a modal interface for user interaction, such as confirmations or forms. It's built upon Radix UI primitives.

### Components

- `Dialog`: The root component that controls the dialog's state.
- `DialogTrigger`: An element that opens the dialog when clicked.
- `DialogPortal`: Renders the dialog into a portal for better DOM management.
- `DialogOverlay`: The backdrop that dims the underlying content.
- `DialogContent`: The main container for dialog content.
- `DialogHeader`: A dedicated section for dialog titles and descriptions.
- `DialogFooter`: A section for action buttons within the dialog.
- `DialogTitle`: The title of the dialog.
- `DialogDescription`: A descriptive text for the dialog.
- `DialogClose`: A button to explicitly close the dialog.

### Usage Example

```tsx
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogDescription,
  DialogFooter,
} from "@/components/ui/dialog";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";

function ExampleComponent() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>Show Dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Are you sure absolutely sure?</DialogTitle>
          <DialogDescription>
            This action cannot be undone. This will permanently delete your account and remove your data from our servers.
          </DialogDescription>
        </DialogHeader>
        <div className="grid gap-4 py-4">
          <div className="grid grid-cols-4 items-center gap-4">
            <label htmlFor="name" className="text-right">Name</label>
            <Input id="name" defaultValue="Pedro Duarte" className="col-span-3" />
          </div>
        </div>
        <DialogFooter>
          <Button variant="outline">Cancel</Button>
          <Button>Confirm</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Input

The `Input` component is a standard text input field with customizable styling and behavior.

### Usage Example

```tsx
import { Input } from "@/components/ui/input";

function ExampleComponent() {
  return (
    <Input type="email" placeholder="Enter your email" />
  );
}
```

## Skeleton

The `Skeleton` component is used to display a placeholder for content that is loading, improving the perceived performance of the UI.

### Usage Example

```tsx
import { Skeleton } from "@/components/ui/skeleton";

function ExampleComponent() {
  return (
    <div className="flex items-center space-x-4">
      <Skeleton className="h-12 w-12 rounded-full" />
      <div className="space-y-2">
        <Skeleton className="h-4 w-[250px]" />
        <Skeleton className="h-4 w-[200px]" />
      </div>
    </div>
  );
}
```

## Key Takeaways

- **Consistency**: The UI components ensure a consistent look and feel across the LandeMon application.
- **Accessibility**: Built with accessibility in mind, leveraging established primitives.
- **Customization**: Props allow for easy customization of appearance and behavior.
- **Reusability**: Components are designed to be easily integrated into various parts of the application.
