## Main Application Logic (`App.jsx`)

The `App.jsx` file contains the main application component.  It utilizes React Router for routing, Zustand for state management (through `useAuthStore` and `useThemeStore`), and handles authentication checks.

```jsx
import { Routes, Route, Navigate } from 'react-router-dom'
import { useEffect } from 'react'
import HomePage from './pages/HomePage'
import SignUpPage from './pages/SignUpPage' 
import LoginPage from './pages/LoginPage' 
import SettingsPage from './pages/SettingsPage' 
import ProfilePage from './pages/ProfilePage'
import { useThemeStore } from './store/useThemeStore'
import { useAuthStore } from './store/useAuthStore'; 
import { Loader } from 'lucide-react'
import { Toaster } from 'react-hot-toast'

const App = () => {
  const { authUser, checkAuth, isCheckingAuth, onlineUsers } = useAuthStore();
  const { theme } = useThemeStore();
  useEffect(() => {
    checkAuth();
  }, [checkAuth]);

  if(isCheckingAuth && !authUser) return (
      <div className='flex items-center justify-center h-screen'>
        <Loader className='size-10 animate-spin' />
      </div>
  )

  return (
    <div className='' data-theme={theme}>
      <Navbar />
      <Toaster />
      <Routes>
        <Route path='/' element={authUser ? <HomePage />: <Navigate to='/login' />} />
        {/* ... other routes */}
      </Routes>
    </div> 
  )
}

export default App
```

[Source](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/src/App.jsx)


This code shows the conditional rendering based on authentication status (`authUser` and `isCheckingAuth`).  If the user is not authenticated and the authentication check is still in progress, a loading indicator is displayed. Otherwise, the application renders the navigation bar (`Navbar`), the routing using `react-router-dom`, and a toaster for notifications.  The routes are protected; unauthorized access redirects to the login page.

Vite is used for the build process. The `vite.config.js` file is straightforward:

import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
})
```

[Source](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/vite.config.js)

This configuration uses the `@vitejs/plugin-react` plugin to enable Vite's support for React development.


## Rendering (`main.jsx`)

The `main.jsx` file handles rendering the application into the DOM:

```javascript
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'
import { BrowserRouter } from 'react-router-dom'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>,
)
```

[Source](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/src/main.jsx)

This code uses `createRoot` to render the `App` component within a `BrowserRouter` to enable client-side routing. The `StrictMode` helps in identifying potential issues in the application.
