# Overview

<TOC />

This document provides an overview of the Chat-App-MERN project, a real-time chat application built using the MERN stack (MongoDB, Express.js, React, Node.js).  The project leverages Socket.IO for real-time communication and incorporates user authentication with Google OAuth.  The following sections detail the project structure, dependencies, and key functionalities.

## Project Structure

The project is divided into two main parts: a backend server and a frontend client.

````mermaid
graph TD
    A[chatapp] --> B(backend);
    A --> C(frontend);
    B --> D[src/index.js];
    C --> E[src/main.jsx];
```

*   **chatapp:** The root directory containing the `package.json` file for managing both the frontend and backend dependencies and build processes.
*   **backend:** Contains the server-side logic, using Express.js, MongoDB, and Socket.IO.  [Source](https://github.com/shinymack/Chat-App-MERN/blob/main/backend)
*   **frontend:** Houses the React-based client-side application. [Source](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend)


## Backend Dependencies

The backend (`backend/package.json`) utilizes several crucial packages:

```json
{
  "dependencies": {
    "bcryptjs": "^2.4.3",
    "cloudinary": "^2.5.1",
    "cookie-parser": "^1.4.7",
    "dotenv": "^16.4.7",
    "express": "^4.21.2",
    "express-session": "^1.18.1",
    "jsonwebtoken": "^9.0.2",
    "mongoose": "^8.9.5",
    "passport": "^0.7.0",
    "passport-google-oauth20": "^2.0.0",
    "socket.io": "^4.8.1"
  }
}
```

*   **Express.js:**  Provides a robust framework for building the RESTful API.
*   **Mongoose:**  An Object Data Modeling (ODM) library for interacting with MongoDB.
*   **Passport.js & passport-google-oauth20:**  Handle user authentication using Google OAuth 2.0.
*   **Socket.IO:** Enables real-time bidirectional communication between the server and clients.
*   **bcryptjs:** Used for password hashing.
*   **jsonwebtoken:** Used for creating and verifying JSON Web Tokens (JWTs) for authentication.
*   **cookie-parser:** Parses cookies from HTTP requests.
*   **dotenv:** Loads environment variables from a `.env` file.
*   **cloudinary:** (Likely used for image/file uploading - inferred from the package name).

## Build and Run

The root `package.json` simplifies the build and run process:

```json
{
  "scripts": {
    "build": "npm install --prefix backend && npm install --prefix frontend && npm run build --prefix frontend",
    "start": "npm run start --prefix backend"
  }
}
```

The `build` script first installs dependencies for both the backend and frontend, then builds the frontend application. The `start` script starts the backend server. This separation of concerns ensures clean build and execution.
