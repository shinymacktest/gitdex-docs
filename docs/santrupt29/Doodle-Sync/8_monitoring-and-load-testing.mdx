# Monitoring and Load Testing

This section details the tools and strategies employed for monitoring the performance of Doodle-Sync and conducting load tests to ensure its scalability and reliability.

## Monitoring with Prometheus and Grafana

Doodle-Sync utilizes Prometheus for metrics collection and Grafana for visualization. Prometheus scrapes metrics exposed by each microservice, providing a centralized source of truth for system performance. Grafana then consumes these metrics to create dashboards that offer insights into real-time performance and historical trends.

### Prometheus Configuration

The `prometheus.yml` configuration file defines the targets from which Prometheus should scrape metrics. Each microservice exposes an `/actuator/prometheus` endpoint for Prometheus to query.

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'api-gateway'
    static_configs:
      - targets: ['api-gateway:8765']
    metrics_path: '/actuator/prometheus'

  - job_name: 'game-service'
    static_configs:
      - targets: ['game-service:8082']
    metrics_path: '/actuator/prometheus'

  - job_name: 'drawing-service'
    static_configs:
      - targets: ['drawing-service:8081']
    metrics_path: '/actuator/prometheus'

  - job_name: 'chat-service'
    static_configs:
      - targets: ['chat-service:8080']
    metrics_path: '/actuator/prometheus'

  - job_name: 'word-service'
    static_configs:
      - targets: ['word-service:8085']
    metrics_path: '/actuator/prometheus'

  - job_name: 'user-service'
    static_configs:
      - targets: ['user-service:8084']
    metrics_path: '/actuator/prometheus'

  - job_name: 'score-service'
    static_configs:
      - targets: ['score-service:8083']
    metrics_path: '/actuator/prometheus'
```

### Grafana Provisioning

Grafana is configured to load dashboards from a specified directory. The `dashboard.yml` file sets up a provider named "scribble" that points to the `/etc/grafana/provisioning/dashboards` path for its dashboard definitions.

```yaml
apiVersion: 1
providers:
  - name: scribble
    folder: Scribble
    type: file
    options:
      path: /etc/grafana/provisioning/dashboards
```

## Load Testing with k6

k6 is used to simulate user load and measure the performance of the Doodle-Sync application under stress. The `stroke-load-test.js` script defines the test scenario, focusing on WebSocket connections for real-time drawing synchronization.

### Load Test Script (`stroke-load-test.js`)

This script simulates multiple virtual users (VUs) joining a game room and sending drawing strokes over WebSockets. It defines performance thresholds for key metrics like stroke latency and WebSocket connection errors.

```javascript
import ws   from 'k6/ws';
import http from 'k6/http';
import { check, sleep }  from 'k6';
import { Counter, Trend } from 'k6/metrics';

const strokesSent     = new Counter('strokes_sent');
const strokeLatency   = new Trend('stroke_latency_ms');
const wsConnectErrors = new Counter('ws_connect_errors');

export const options = {
  vus:      8,          // 8 virtual users = 8 concurrent players
  duration: '60s',
  thresholds: {
    stroke_latency_ms:   ['p(99) < 200'],
    'http_req_failed':   ['rate < 0.01'],
    ws_connect_errors:   ['count < 3'],
  },
};

export function setup() {
  const headers = { 'Content-Type': 'application/json' };

  const regPayload = JSON.stringify({
    username: 'loadtest123',
    email:    'load123@test.com',
    password: 'pass123',
  });

  let authData;
  const reg = http.post('http://localhost:8765/user/auth/register', regPayload, { headers });

  if (reg.status === 200 || reg.status === 201) {
    authData = reg.json();
  } else {
    console.log(`Registration skipped (${reg.status}). Logging in...`);
    const loginPayload = JSON.stringify({ username: 'loadtest123', password: 'pass123' });
    const login = http.post('http://localhost:8765/user/auth/login', loginPayload, { headers });

    if (login.status !== 200) {
      console.error(`Login failed: ${login.status} — ${login.body}`);
      throw new Error('Could not authenticate loadtest user');
    }
    authData = login.json();
  }

  const token    = authData.token;
  const userId   = authData.userId;

  if (!token || !userId) {
    throw new Error(`Auth incomplete — token: ${token}, userId: ${userId}`);
  }

  const roomPayload = JSON.stringify({
    userId,
    username:       'loadtest123',
    maxPlayers:     8,
    totalRounds:    2,
    drawTimeSeconds: 90,
  });

  const room = http.post('http://localhost:8765/game/room', roomPayload, {
    headers: {
      'Content-Type':  'application/json',
      'Authorization': `Bearer ${token}`,
    },
  });

  if (room.status !== 200 && room.status !== 201) {
    console.error(`Room creation failed: ${room.status} — ${room.body}`);
    throw new Error('Could not create room');
  }

  const roomCode = room.json('roomCode');
  console.log(`Setup complete — room: ${roomCode}`);

  return { roomCode, token };
}

export default function (data) {
  const { roomCode, token } = data;
  const vuId = __VU;

  // Fetch a one-time WS auth ticket before connecting
  const ticketRes = http.post('http://localhost:8765/user/auth/ws-ticket', null, {
    headers: { 'Authorization': `Bearer ${token}` },
  });

  if (ticketRes.status !== 200) {
    console.error(`Ticket fetch failed: ${ticketRes.status} — ${ticketRes.body}`);
    wsConnectErrors.add(1);
    return;
  }

  const ticket = ticketRes.json('ticket');
  const url = `ws://localhost:8081/ws/websocket?ticket=${ticket}`;

  const res = ws.connect(url, {}, function (socket) {

    socket.on('open', function () {
      socket.send('CONNECT\naccept-version:1.2\nheart-beat:0,0\n\n\x00');
    });

    socket.on('message', function (msg) {

      if (msg.startsWith('CONNECTED')) {

        socket.send(
          'SUBSCRIBE\n' +
          'id:sub-' + vuId + '\n' +
          'destination:/topic/room.' + roomCode + '.canvas\n\n\x00'
        );

        let seq = 0;
        socket.setInterval(function () {
          const start = Date.now();
          const stroke = JSON.stringify({
            playerId:    'vu-' + vuId,
            roomCode:    roomCode,
            x1: Math.random() * 800,
            y1: Math.random() * 600,
            x2: Math.random() * 800,
            y2: Math.random() * 600,
            color:       '#000000',
            width:       5,
            isEraser:    false,
            timestamp:   start,
            sequenceNum: ++seq,
          });

          socket.send(
            'SEND\n' +
            'destination:/app/room.' + roomCode + '.stroke\n' +
            'content-type:application/json\n' +
            '\n' + stroke + '\x00'
          );

          strokesSent.add(1);
          strokeLatency.add(Date.now() - start);
        }, 100);  // every 100ms = 10 strokes/sec

        socket.setTimeout(function () {
          socket.close();
        }, 58000);
      }
    });

    socket.on('error', function (e) {
      console.error('WS error: ' + (e.message || e));
      wsConnectErrors.add(1);
    });

    socket.on('close', function () {
      // no-op: k6 will end the VU iteration
    });
  });

  check(res, { 'WS connected': (r) => r && r.status === 101 });
}
```

## Key Takeaways

- **Centralized Metrics**: Prometheus aggregates metrics from all services, offering a unified view of system health.
- **Real-time Visualization**: Grafana dashboards provide immediate insights into performance, enabling quick identification of bottlenecks.
- **Scalability Testing**: k6 load tests validate the application's ability to handle concurrent users and high message volumes, particularly for real-time features like drawing.
- **Performance Benchmarking**: Defined thresholds in k6 tests ensure that critical performance indicators like latency and error rates remain within acceptable limits.
