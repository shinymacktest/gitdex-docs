# Frontend Overview & Core

This section provides a high-level overview of the chat application's frontend, built with React and Vite. It covers the project setup, development configurations, and the core application structure, including routing and initial components.

## Project Setup and Configuration

The frontend is a standard React + Vite project, offering a fast development experience with Hot Module Replacement (HMR).

## Application Structure & Routing (`App.jsx`)

The `App.jsx` file serves as the main entry point for the React application, handling global state, authentication checks, and routing.

```jsx
// frontend/src/App.jsx snippet
import Navbar from './components/Navbar'
import { Routes, Route, Navigate } from 'react-router-dom'
import { useEffect } from 'react'
import HomePage from './pages/HomePage'
import SignUpPage from './pages/SignUpPage'
// ... other page imports
import { useAuthStore } from './store/useAuthStore';

const App = () => {
  const { authUser, checkAuth, isCheckingAuth, onlineUsers } = useAuthStore();
  // ... theme store
  useEffect(() => {
    checkAuth(); // Check user authentication status on component mount
  }, [checkAuth]);

  if(isCheckingAuth && !authUser) return (
      <div className='flex items-center justify-center h-screen'>
        <Loader className='size-10 animate-spin' />
      </div>
  )

  return (
    <div className='' data-theme={theme}>
      <Navbar />
      <Toaster /> {/* Global toast notifications */}
      <Routes>
        <Route path='/' element={authUser ? <HomePage />: <Navigate to='/login' />} />
        <Route path='/signup' element={ !authUser ? <SignUpPage />: <Navigate to='/' />} />
        <Route path='/login' element={!authUser ? <LoginPage />: <Navigate to='/' />} />
        <Route path='/settings' element={<SettingsPage />} />
        <Route path='/profile' element={authUser ? <ProfilePage />: <Navigate to='/login' />} />
      </Routes>
    </div>
  )
}

export default App
```
[Source: frontend/src/App.jsx](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/src/App.jsx)

### Key Responsibilities:
*   **Authentication Check**: On initial load, `checkAuth()` verifies the user's login status. A loading spinner is displayed during this check.
*   **Global State**: Utilizes `zustand` (`useAuthStore`, `useThemeStore`) for managing application-wide state like authenticated user data, online users, and theme.
*   **Routing**: Employs `react-router-dom` to define routes. Protected routes (`/`, `/profile`) redirect unauthenticated users to `/login`, while auth-related routes (`/signup`, `/login`) redirect authenticated users to `/`.
*   **Global Components**: Integrates a `Navbar` and `react-hot-toast` for notifications across the application.

## Core Components

### AuthImagePattern (`AuthImagePattern.jsx`)
This component provides a decorative pattern for authentication pages, enhancing the visual appeal.

```jsx
const AuthImagePattern = ({ title, subtitle }) => {
    return (
      <div className="hidden lg:flex items-center justify-center bg-base-200 p-12">
        <div className="max-w-md text-center">
          <div className="grid grid-cols-3 gap-3 mb-8">
            {[...Array(9)].map((_, i) => (
              <div
                key={i}
                className={`aspect-square rounded-2xl bg-primary/10 ${
                  i % 2 === 0 ? "animate-pulse" : ""
                }`}
              />
            ))}
          </div>
          <h2 className="text-2xl font-bold mb-4">{title}</h2>
          <p className="text-base-content/60">{subtitle}</p>
        </div>
      </div>
    );
  };

export default AuthImagePattern;
```
[Source: frontend/src/components/AuthImagePattern.jsx](https://github.com/shinymack/Chat-App-MERN/blob/main/frontend/src/components/AuthImagePattern.jsx)

It dynamically generates a grid of pulsing squares and displays a title and subtitle, typically used alongside login or signup forms.
