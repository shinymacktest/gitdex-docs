# Overview & Getting Started
<TOC />

This section provides a high-level introduction to the Chat-App-MERN project, detailing its core components, technology stack, and initial steps to get the application running. The project is a real-time chat application built using the MERN stack (MongoDB, Express.js, React, Node.js) and enhanced with Socket.IO for live communication.

The application allows users to register, log in, manage friends, and engage in one-on-one real-time conversations. It leverages modern web technologies to provide a responsive and interactive user experience.

## Project Structure

The Chat-App-MERN project is organized into two main parts: `backend` and `frontend`, encapsulated within a root directory. This monorepo-like structure facilitates cohesive development and deployment.

````mermaid
graph TD
    A[Chat-App-MERN Project] --> B(Backend)
    A --> C(Frontend)

    B --> D{Node.js & Express.js}
    B --> E{MongoDB & Mongoose}
    B --> F{Socket.IO Server}
    B --> G{Authentication (JWT, Passport.js)}

    C --> H{React & Vite}
    C --> I{Socket.IO Client}
    C --> J{State Management (Zustand)}
    C --> K{UI Frameworks (TailwindCSS, DaisyUI)}
```

### Key Directories

*   **`backend/`**: Contains the server-side logic, API endpoints, database interactions, and Socket.IO server implementation.
*   **`frontend/`**: Houses the client-side React application, user interface components, and real-time client-side communication.
*   **`package.json` (root)**: Defines scripts for building and starting the entire application, orchestrating both backend and frontend processes.

## Technology Stack

The Chat-App-MERN project utilizes a robust set of technologies for both its server-side and client-side components.

### Backend Technologies

The backend is built with Node.js and Express, providing a powerful and flexible foundation.

*   **Express.js**: Web framework for building RESTful APIs.
*   **MongoDB & Mongoose**: NoSQL database for data storage, with Mongoose as an ODM for object modeling.
*   **Socket.IO**: Enables real-time, bidirectional, event-based communication between the server and clients.
*   **JWT & Passport.js**: For secure user authentication and authorization, including Google OAuth2.0 integration.
*   **Bcryptjs**: Password hashing library for secure user credentials.
*   **Cloudinary**: Cloud-based image and video management for handling user profile pictures.
*   **Dotenv**: Manages environment variables for configuration.
*   **Nodemon**: Development tool for automatic server restarts on file changes.

The backend entry point `backend/src/index.js` demonstrates the server setup, middleware configuration, and routing:

<details>
<summary><code>backend/src/index.js</code></summary>

```javascript
import express from "express";
import cors from "cors";
import authRoutes from "./routes/auth.route.js";
import messageRoutes from "./routes/message.route.js";
import friendRoutes from "./routes/friend.route.js";

import path from "path";

import dotenv from "dotenv";
import cookieParser from "cookie-parser";

import { connectDB } from "./lib/db.js";
import { app, server } from "./lib/socket.js";

import session from "express-session";
import passport from "passport";
import { configurePassport } from "./lib/passport.config.js";

const __dirname = path.resolve();
dotenv.config();

configurePassport();

app.use(cookieParser());
app.use(express.json({limit : '2mb'}));
app.use(express.urlencoded({ limit: '2mb', extended: true }));
app.use(cors({
    origin: "http://localhost:5173",
    credentials: true,
}));

app.use(session({
    secret: process.env.SESSION_SECRET,
    resave: false,
    saveUninitialized: false,
    cookie: {
        secure: process.env.NODE_ENV === "production", // true in production (HTTPS)
        httpOnly: true,
        maxAge: 7 * 24 * 60 * 60 * 1000
        // sameSite: 'lax' // or 'none' if backend and frontend are on different domains in prod
    }
}));

app.use(passport.initialize());
app.use(passport.session());

app.use("/api/auth", authRoutes );
app.use("/api/messages", messageRoutes );
app.use("/api/friends", friendRoutes);


const PORT = process.env.PORT;
if(process.env.NODE_ENV === "production"){
    app.use(express.static(path.join(__dirname, "../frontend/dist")));

    app.get("*" , (req, res) => {
        res.sendFile(path.join(__dirname,"../frontend", "dist","index.html"));
    })
}

server.listen(PORT, () => {
    console.log("server is running on PORT: " + String(PORT));
    connectDB();
});
```

### Frontend Technologies

The frontend is a modern single-page application built with React and Vite.

*   **React**: A declarative JavaScript library for building user interfaces.
*   **Vite**: A fast build tool that significantly improves the frontend development experience.
*   **Socket.IO Client**: Establishes a WebSocket connection with the backend for real-time updates.
*   **React Router DOM**: For client-side routing and navigation.
*   **Zustand**: A lightweight, fast, and scalable state management solution for React.
*   **TailwindCSS & DaisyUI**: Utility-first CSS framework and a component library for rapid UI development.
*   **Axios**: Promise-based HTTP client for making API requests.
*   **Lucide React & React Icons**: Icon libraries for various UI elements.
*   **React Hot Toast**: For displaying elegant and responsive notifications.

## Getting Started

To set up and run the Chat-App-MERN project locally, follow these steps:

### 1. Clone the Repository

First, clone the project repository to your local machine:

```bash
git clone https://github.com/shinymack/Chat-App-MERN.git
cd Chat-App-MERN
```

### 2. Install Dependencies

The project uses a root `package.json` to streamline the installation process for both backend and frontend.

<details>
<summary><code>package.json</code> (root)</summary>

```json
{
  "name": "chatapp",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "build" : "npm install --prefix backend && npm install --prefix frontend && npm run build --prefix frontend",
    "start" : "npm run start --prefix backend"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "description": ""
}
```
[Source](https://github.com/shinymack/Chat-App-MERN/blob/main/package.json)
</details>

Run the following command from the project root directory:

```bash
npm install
# This will install dependencies for both backend and frontend based on the root build script.
# Alternatively, you can navigate into each sub-directory and run npm install manually:
# cd backend && npm install && cd ..
# cd frontend && npm install && cd ..
```

### 3. Environment Variables

Create a `.env` file in the `backend` directory and configure the necessary environment variables. These typically include database connection strings, API keys, and secret keys for authentication.

```
# Example .env for backend/
MONGO_URI=mongodb+srv://<username>:<password>@cluster0.abcde.mongodb.net/chat-app?retryWrites=true&w=majority
PORT=5000
JWT_SECRET=your_jwt_secret_key
SESSION_SECRET=your_session_secret_key
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
CLIENT_URL=http://localhost:5173
```

### 4. Run the Application

Once dependencies are installed and environment variables are configured, you can start the application:

#### Development Mode (Recommended for Local Development)

To run the backend and frontend separately for development with hot-reloading:

```bash
# In the project root, open two separate terminal windows:

# Terminal 1: Start the backend server
cd backend
npm run dev

# Terminal 2: Start the frontend development server
cd frontend
npm run dev
```

The frontend will typically be accessible at `http://localhost:5173` and the backend at `http://localhost:5000` (or your configured `PORT`).

#### Production Mode (Using Root `package.json` scripts)

The root `package.json` provides scripts for building and starting the entire application in a more production-like manner.

1.  **Build the frontend**:
    ```bash
    npm run build --prefix frontend
    # Or, if you want to use the root build script which handles both installs and frontend build:
    # npm run build
    ```
    This command will compile the React application into static files in `frontend/dist`.

2.  **Start the backend (which also serves the built frontend in production mode)**:
    ```bash
    npm start # This script from root package.json will run "npm run start --prefix backend"
    ```
    The backend server will then serve the static frontend files when accessed in production mode, as configured in `backend/src/index.js`.

The application should now be running, and you can access it via your web browser (typically `http://localhost:5000` or the `PORT` specified in your backend's `.env`).
