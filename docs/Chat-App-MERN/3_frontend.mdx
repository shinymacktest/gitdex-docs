## Project Overview

This document details the frontend development of a React application, built using Vite.  The application utilizes various libraries for routing, styling, state management, and user interface components.  The core functionality involves user authentication, navigation between different pages (Home, Sign Up, Login, Settings, Profile), and potentially real-time features (implied by the presence of `socket.io-client`).

## Core Application Logic (`App.jsx`)

The `App.jsx` file is the main entry point of the application.  It handles routing, authentication checks, and theme management.

```jsx
import Navbar from './components/Navbar'
import { Routes, Route, Navigate } from 'react-router-dom'
import { useEffect } from 'react'
import HomePage from './pages/HomePage'
import SignUpPage from './pages/SignUpPage' 
import LoginPage from './pages/LoginPage' 
import SettingsPage from './pages/SettingsPage' 
import ProfilePage from './pages/ProfilePage'


import { useThemeStore } from './store/useThemeStore'
import { useAuthStore } from './store/useAuthStore'; 
import { Loader } from 'lucide-react'
import { Toaster } from 'react-hot-toast'

const App = () => {
  const { authUser, checkAuth, isCheckingAuth, onlineUsers } = useAuthStore();
  const { theme } = useThemeStore();
  useEffect(() => {
    checkAuth();
  }, [checkAuth]);

  // ... rest of the code
}

export default App
```

This code uses `react-router-dom` for routing, `zustand` through custom hooks (`useAuthStore`, `useThemeStore`) for state management (authentication and theme), and `react-hot-toast` for notifications.  The `useEffect` hook calls `checkAuth` on mount to determine the user's authentication status.  Conditional rendering ensures that users are redirected appropriately based on authentication.

### Routing Configuration

The application uses React Router to manage navigation between different pages.  The routing logic is defined within the `App` component.

```jsx
<Routes>
  <Route path='/' element={authUser ? <HomePage />: <Navigate to='/login' />} />
  <Route path='/signup' element={ !authUser ? <SignUpPage />: <Navigate to='/' />} />
  <Route path='/login' element={!authUser ? <LoginPage />: <Navigate to='/' />} />
  <Route path='/settings' element={<SettingsPage />} />
  <Route path='/profile' element={authUser ? <ProfilePage />: <Navigate to='/login' />} />
</Routes>
```

This code defines routes for the home page (`/`), sign-up (`/signup`), login (`/login`), settings (`/settings`), and profile (`/profile`).  Conditional rendering using `authUser` ensures that only authenticated users can access protected routes like `/` and `/profile`.


## Styling (`index.css`)

The application uses Tailwind CSS and DaisyUI for styling, with a custom Google Font.

```css
@import url('https://fonts.googleapis.com/css2?family=Chivo:ital,wght@0,100..900;1,100..900&display=swap');
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
    body {
        @apply font-chivo;
    }
}
```

This imports the Chivo font and then includes the Tailwind CSS base, components, and utilities.  A base layer is added to apply the Chivo font to the body.

import { defineConfig } from 'vite'

export default defineConfig({

})
